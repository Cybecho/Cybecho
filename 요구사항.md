## GitHub Actions 전환 가이드

`.github/workflows/readme-rotation.yml`로 크론 기반 자동화를 대체합니다.

### 스케줄
- `*/30 * * * *` (readme 갱신)
- `0 */12 * * *` (imgList 갱신 + readme 갱신)

### 수동 실행 (workflow_dispatch)
- 수동 실행 시 `writeUrl.py`를 먼저 실행해서 `imgList.txt`를 갱신하고,
- 이어서 `randomImg.sh`를 실행해 README 갱신까지 한 번에 반영합니다.

### 권장 실행 동작
1. 12시간 주기일 때만 `writeUrl.py` 실행
2. `randomImg.sh` 실행
3. 변경분이 있을 때만 commit/push

### 비고
- 현재 `spooky` 스크립트는 기본 자동화에서 제외되며 필요 시 수동 워크플로 또는 수동 실행에서만 사용하세요.

### spooky 이스터에그
- `spooky/randomImgSpooky.sh`는 `.github/workflows/spooky-rotation.yml`에서 별도 실행됩니다.
- 실행 시각: `KST 04:44` (GitHub Actions cron은 UTC 기준이므로 `44 19 * * *`).
- 수동 실행도 `workflow_dispatch`로 가능합니다.
  - UI: Actions → rotate-spooky-readme → Run workflow에서 `force_run` 체크 후 실행
